## 1. プロジェクト概要

### 1.1 プロダクトビジョン
クライアント向けAI導入ROI試算ツール。初回商談（30分）の場で「御社のAI導入ROI」を即座にシミュレーションし、提案の説得力を劇的に高める**営業・コンサルティングツール**。

### 1.2 戦略的位置づけ
1. **商談クロージングツール**：初回商談でクライアントに「自社のAI導入ROI」を即座に見せ、次回アポ・受注を引き出す
2. **ポートフォリオ**：「AI変革アーキテクト」としての技術力・ビジネス理解の証明
3. **差別化要因**：他のAIコンサルタントが持っていない「定量化の武器」

### 1.3 ターゲットユーザー
- **プライマリ**：オーナー自身（商談中に操作する人）
- **セカンダリ**：クライアントの経営層・意思決定者（画面を見る人）
- **ターシャリ**：クライアントの担当者（後日URLでアクセスする人）

## 2. ユースケース

### 2.1 ユースケース優先度

| 優先度 | シーン | 説明 | UI品質 |
|--------|--------|------|--------|
| **P0** | 初回オンライン商談 | 画面共有しながらリアルタイムでROI試算を見せる | プレゼンテーション品質 |
| **P0** | 提案書への埋め込み | 試算結果のスクリーンショットを提案書（PPT）に貼る | 印刷品質 |
| **P1** | クライアントのセルフ利用 | URLにアクセスし自社パラメータを入力して試す | 一般Webアプリ品質 |
| **P2** | 効果測定ベースライン | 受注後プロジェクトの効果測定の基準値として活用 | 機能品質 |

### 2.2 商談フロー（シーン1の詳細）

```
0-5分   : 自己紹介・ヒアリング（ツール未使用）
5-15分  : 課題共有 → 業種選択、AI適用領域の提示
15-25分 : ROI提示 → パラメータ入力、リアルタイムでグラフが動く ★クライマックス
25-30分 : クロージング → 「詳細レポートをお送りします」
```

## 4. 非機能要件

### 4.1 パフォーマンス
- **初期表示**：3秒以内（商談中の画面切り替えでストレスを与えない）
- **計算レスポンス**：スライダー操作に対して60fps以上でグラフ更新
- **オフライン対応**：ネットワーク不安定な環境でもコア機能が動作すること

### 4.2 UI/UX品質
- **プレゼンテーション対応**：1280px以上の画面で見栄えするレイアウト
- **スクリーンショット対応**：白背景モードで印刷品質の出力が可能
- **ブランディング**：「AI変革アーキテクト」としてのプロフェッショナルな印象
- **30秒ルール**：経営層が画面を見て30秒で要点を理解できる情報設計

### 4.3 信頼性
- **計算の正確性**：ROI計算ロジックにユニットテスト必須
- **前提条件の透明性**：すべての試算結果に前提条件と免責事項を表示
- **データソースの明示**：ベンチマークデータの出典を表示可能にする

### 4.4 セキュリティ
- クライアントの入力データはサーバーに送信しない（フロントエンド完結）
- 個人情報や機密情報の入力フィールドを設けない

### 4.5 保守性
- TypeScriptによる型安全な実装
- コンポーネントベースの設計で機能追加が容易
- ベンチマークデータの更新が定数変更のみで可能

## 5. ROIモデル要件（概要）

詳細は `docs/roi-model-design.md` を参照。

### 5.1 入力パラメータ
- **第1層（必須）**：業種、従業員数、売上規模、AI導入対象業務（10カテゴリ）
- **第2層（任意）**：月間コスト、従事人数、IT投資額、AI活用レベル

### 5.2 計算モデル
- **Model A**：コスト削減型
- **Model B**：売上増加型
- **Model C**：生産性向上型（FTE換算）
- **複合計算**：対象業務に応じてA/B/Cを組み合わせ

### 5.3 出力指標
| 優先度 | 指標 |
|--------|------|
| P0 | 3年間累計ROI（%）、年間コスト削減額、投資回収期間（月）、年間売上増加額 |
| P1 | FTE換算、5年間累計効果額 |
| P2 | NPV（正味現在価値） |

### 5.4 シナリオ
保守的 / 標準 / 楽観的 の3シナリオを同時表示。

## 6. 競合分析

### 6.1 市場の現状

市場調査の結果、以下が判明：

**既存ツールの分類：**
1. **汎用ROI計算機**（AI4SP、BoldDesk等）— 簡易的な入力で概算ROIを表示。過度に単純化されたモデル
2. **ベンダー特化型**（Microsoft Copilot ROI Calculator、IFS等）— 自社プロダクトの導入効果に特化。汎用性なし
3. **ドメイン特化型**（Cycode: AppSec、eGain: 顧客サービス等）— 特定業務領域に限定
4. **コンサルファーム**（McKinsey、Accenture、Deloitte等）— リサーチ・フレームワークは公開するが、**ツール自体は有料コンサルティングの中でのみ提供**
5. **日本市場** — Excelテンプレートやガイド記事はあるが、**インタラクティブなWebベースのAI ROIシミュレーターは存在しない**

### 6.2 既存ツールの共通課題
| 課題 | 説明 |
|------|------|
| **過度な単純化** | 「時間削減 × 単価」のみの計算。導入コスト、採用カーブ、組織変革コストを考慮しない |
| **TCOモデリングの欠如** | データ準備、ガバナンス、トレーニング、メンテナンスコストを含まない |
| **リスク調整なし** | 感度分析やシナリオベースのリスクモデリングがほぼ皆無 |
| **静的な前提** | 固定的な採用率を前提、ランプアップ期間や逓減/逓増効果を非モデル化 |
| **単一ドメイン** | 各ツールが特定ベンダーエコシステムか狭いユースケースに紐付き |
| **日本語対応なし** | 日本語のインタラクティブツールは市場に存在しない |

### 6.3 本ツールの差別化ポイント

| # | 差別化要素 | 詳細 |
|---|-----------|------|
| 1 | **マルチ業種×マルチ業務対応** | 10業種×10業務カテゴリの組み合わせで汎用的に試算 |
| 2 | **3シナリオ比較** | 保守的/標準/楽観的を同時表示。既存ツールのほとんどが単一シナリオのみ |
| 3 | **インタラクティブ感度分析** | スライダー操作でリアルタイムにROIが変動。既存ツールにはほぼない |
| 4 | **実績ベースのキャリブレーション** | 5社のAI導入実績によるベンチマーク補正。汎用ツールにはない精度 |
| 5 | **コンサルタント営業に最適化されたUX** | 「商談の流れ」に沿ったステッパーUI。既存ツールは「計算機」止まり |
| 6 | **プレゼンテーション品質のビジュアル** | 画面共有・スクリーンショットで映えるデザイン。Excelやシンプルな計算機との決定的な差 |
| 7 | **日本市場初のインタラクティブWebツール** | 日本語でのインタラクティブAI ROIシミュレーターは競合不在 |
| 8 | **前提条件の完全な透明性** | すべての数値の根拠とデータソースを明示。コンサルとしての信頼性を担保 |

## 7. 成功指標（KPI）

| KPI | 測定方法 | 目標値 |
|-----|---------|--------|
| 商談→次回アポ獲得率 | ツール使用商談/次回アポ獲得数 | 70%以上 |
| 提案書作成時間 | ツール使用前後の比較 | 50%短縮 |
| ツール利用頻度 | 商談でのツール使用率 | 全商談の80%以上 |
| クライアント反応 | 質問・深掘り・社内共有リクエストの発生 | 定性評価 |

### 究極の成功指標
> 「あのROI試算ツール、うちの経営会議でも使いたいんですが」とクライアントに言わせること

## 9. 制約事項

### 9.1 スコープ制約
- MVPは「初回商談で使える」最小限の機能セットに限定
- モバイル対応はスコープ外（PC画面共有が主要利用シーン）
- 多言語対応はスコープ外（日本語のみ）

### 9.2 データ制約
- ベンチマークデータは公開レポートおよびオーナー実績に基づく推定値
- リアルタイムの外部データ連携はしない

### 9.3 時間制約
- 2026年4月のフリーランス独立までにMVPを完成させる
