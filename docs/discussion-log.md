## 1.1 ユースケースの具体化

### 優先度決定

| 優先度 | シーン | 決定理由 |
|--------|--------|---------|
| **P0** | シーン1：初回オンライン商談で画面共有しながらROIを見せる | 主戦場。ツールの存在意義そのもの |
| **P0** | シーン2：提案書にスクリーンショットとして貼る | シーン1と一体。商談後フローで必須 |
| **P1** | シーン3：クライアント自身がURLにアクセスして試す | 検討フェーズで効果大。MVP後対応可 |
| **P2** | シーン4：受注後の効果測定ベースライン | 受注後の話。優先度低 |

### 【Consultant】商談フロー設計
商談30分の具体的な流れ：
1. **冒頭（0-5分）**：自己紹介・ヒアリング → ツール未使用
2. **課題共有（5-15分）**：業種選択し、AI適用領域を提示
3. **ROI提示（15-25分）**：パラメータ入力、目の前で数字が動く演出 ← **クライマックス**
4. **クロージング（25-30分）**：「詳細レポートをお送りします」

### 【UX】UI品質レベル定義
- シーン1（画面共有）：プレゼンテーション品質 — 大フォント、高コントラスト、アニメーション
- シーン2（スクショ）：印刷品質 — 白背景、高情報密度でも可読、ブランディング要素
- シーン3（セルフサービス）：一般Webアプリ品質 — 説明テキスト充実、バリデーション

### 【UX】Excelとの決定的差別化3要素
1. **動的ビジュアライゼーション** — スライダー操作でリアルタイムグラフ変動
2. **プロフェッショナルなブランディング** — 「ここまでやるのか」という印象
3. **ストーリーテリング** — 入力→分析→結論のガイド付きフロー

## 1.2 ROIモデルの設計議論

### 【Consultant】基本設計原則
**最大のリスクは「嘘っぽい数字を出すこと」**
- 経営層は数字のプロ。楽観的すぎる数字は逆効果
- 保守的な数字でも根拠が明確なら信頼される
- すべての前提条件を透明にする

### 入力パラメータ設計：二段構え方式
**第1層（必須 — 商談中に聞ける情報）**：業種、従業員数、売上規模、AI導入対象業務
**第2層（任意 — 精度向上用）**：月間コスト、従事人数、IT投資額

設計原則：商談中に聞けない情報は必須にしない

### 計算モデル
- **Model A（コスト削減型）**：対象業務コスト × AI自動化率 × 業種補正係数
- **Model B（売上増加型）**：現在売上 × AI効果係数 × 業種別成長率
- **Model C（生産性向上型）**：FTE換算による時間削減の金額換算
- **複合表示**：選択業務に応じてA/B/Cを組み合わせ

### 【Consultant】オーナー実績の組み込み方
- **推奨**：ベンチマークのキャリブレーション素材 + ケーススタディ表示
- **非推奨**：計算式への直接組み込み（Amazon固有環境の恩恵が大きく、一般企業に適用するとミスリードの危険）

### 【Consultant】前提条件の透明性
- すべての試算結果に前提条件を明示
- ベンチマークデータのソースを表示可能にする
- シナリオ別の想定をドキュメント化

## 1.3 機能候補の議論

### MVP機能セット決定

| フェーズ | 機能 |
|---------|------|
| **MVP** | F1（入力UI）、F2（ダッシュボード）、F3（シナリオ比較）、F4（業種別テンプレート）、F9（感度分析）、F11（導入あり/なし比較） |
| **v1.1** | F5（AI適用領域マッピング）、F6（ロードマップ）、F7（PDF出力）、F8（ケーススタディ）、F12（共有リンク） |
| **v2.0** | F10（LLM考察生成） |

### 【Consultant】F3（シナリオ比較）の重要性
コンサルタントの信頼性を守る防御線。「保守的に見てもこれだけの効果」と言えることが長期信頼につながる。

### 【UX】UIフローとしての機能配置
```
[入力:F1] → [結果:F2] → [3シナリオ比較:F3] → [感度分析:F9] → [将来推移:F11]
```
この流れ自体が商談プレゼンの構成。ステッパーUIで商談の流れに沿って進行。

### 【Architect】技術的判断
MVP機能セットはすべてフロントエンドで完結可能。外部API依存ゼロのSPAとしてビルド。

## 1.4 競合・類似ツール分析

### 【Consultant】市場調査結果（詳細）

**既存ツールの5分類：**
1. **汎用ROI計算機**（AI4SP、BoldDesk等）— 簡易入力で概算。過度に単純化
2. **ベンダー特化型**（Microsoft Copilot ROI Calculator、IFS等）— 自社製品導入効果のみ
3. **ドメイン特化型**（Cycode: AppSec、eGain: 顧客サービス等）— 特定業務限定
4. **コンサルファーム**（McKinsey、Accenture、Deloitte）— フレームワークは公開するが**ツールは有料コンサルの中でのみ提供**
5. **日本市場** — Excelテンプレート・ガイド記事はあるが**インタラクティブWebツールは皆無**

**既存ツールの共通課題：**
- 過度な単純化（「時間削減×単価」のみ）
- TCOモデリングの欠如
- 感度分析・シナリオ比較がほぼない
- 静的な前提（ランプアップ期間を非考慮）
- 日本語インタラクティブツール不在

### 差別化ポイント（調査結果ベース）
1. **マルチ業種×マルチ業務**：10業種×10業務の組み合わせ対応（既存は単一ドメインが主）
2. **3シナリオ比較**：保守的/標準/楽観的の同時表示（既存はほぼ単一シナリオ）
3. **インタラクティブ感度分析**：スライダーでリアルタイム変動（既存にはほぼない）
4. **実績キャリブレーション**：5社AI導入実績による補正
5. **コンサルタント営業最適化UX**：商談フローに沿ったステッパーUI
6. **プレゼンテーション品質ビジュアル**：画面共有・スクショで映えるデザイン
7. **日本市場初のインタラクティブWebツール**：競合不在
8. **前提条件の完全な透明性**：根拠とデータソースを明示

### 【Consultant】競合分析の結論
> **「コンサルファームは有料コンサル内でしかツールを見せない。我々はそれを初回商談で無料で見せる。これだけで圧倒的な差別化になる。」**
> 加えて、日本語のインタラクティブAI ROIシミュレーターは市場に存在しない。ファーストムーバーとして大きなアドバンテージ。
